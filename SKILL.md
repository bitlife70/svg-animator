# SVG Animator Skill

## Overview
이 스킬은 단순한 다이어그램을 넘어, 프로페셔널하고 현대적인 느낌의 SVG 애니메이션을 생성하기 위한 전문 지침을 제공합니다. 특히 데이터 흐름과 시스템 인포그래픽에 최적화되어 있습니다.

## 핵심 가이드라인

### 1. 색상 및 스타일 (Tailwind CSS)
- 모든 색상은 [tailwind-colors.md](references/tailwind-colors.md)에 정의된 팔레트를 사용합니다.
- 배경은 주로 `Slate-900` (#0f172a)을 사용하고, 데이터 노드와 흐름에 강조색(`Sky-400`, `Cyan-400`)을 적용합니다.
- 선(Stroke)은 둥근 끝(round) 처리를 기본으로 하여 부드러운 느낌을 줍니다.

### 2. 고급 레이아웃 및 라우팅 (Advanced Routing)
- **직각 라우팅 (Orthogonal Routing)**: 시스템 아키텍처 다이어그램에서는 곡선보다 직각(90도)으로 꺾인 직선 경로를 우선적으로 사용합니다. 이는 시각적으로 더 정돈된 구조를 제공합니다.
- **선 겹침 방지 (No Overlap)**: 경로(Path)들이 서로 교차하거나 겹치지 않도록 좌표를 세밀하게 설계합니다. 
- **선 교차 최소화 (Minimize Crossing)**: 경로(Path)들이 서로 교차하는 것을 최소화합니다. 
- **박스 우회**: 연결선이 다른 컨테이너 박스의 내용을 가로지르지 않도록 박스 외부를 우회하여 라우팅합니다.

### 3. 가독성 및 텍스트 관리 (Readability & Text Management)
- **일반 굵기 사용 (Normal Weight for Labels)**: 화살표 라인 위의 흐름 설명 라벨에는 `font-weight: bold`를 사용하지 않고 일반 굵기(`normal`)를 유지하여 시각적 노이즈를 줄입니다.
- **제로 스트로크 원칙 (Zero-Stroke on Text)**: 텍스트 요소에는 절대 `stroke` 속성을 부여하지 않습니다. 
- **텍스트 최상단 렌더링 (Text-on-Top)**: 모든 배경 박스와 연결선(Path)보다 코드상 나중에 배치하여 가려짐을 차단합니다.

### 4. 전문적 아이콘 및 섹션 관리
- **아이콘 자원 활용**: 아이콘 선택 시 [svgrepo.com](https://www.svgrepo.com/) 등을 활용하여 다양하고 직관적인 고품질 SVG 아이콘을 사용합니다.
- **아이콘 누락 금지 (Icon Restoration)**: "Unstructured Content Sources"와 같은 하단 섹션에서도 상단과 동일한 수준의 전문적 아이콘(`symbol`)을 반드시 배치하여 시각적 균형을 맞춥니다.
- **배경 대비**: 텍스트와 배경색이 동일하여 보이지 않는 현상을 방지하기 위해, 어두운 배경(`4c1d95`) 위에는 반드시 밝은 색(`white`, `ddd6fe`)의 텍스트를 사용합니다.

### 5. 정밀한 앵커링 및 포션 파악 (Proportional Anchoring)
- **포션 기반 설계**: 원본 이미지의 구성 요소별 비율(예: 상단 15%, 중단 65%, 하단 20%)을 먼저 파악하여 `viewBox` 내 좌표를 할당합니다.
- **에지 투 에지(Edge-to-Edge)**: 모든 화살표와 연결선은 박스의 **경계선(Edge) 외부**에서 시작하고 끝나야 합니다. 박스 내부(Center)에서 선이 시작되어 도형을 가로지르는 실수를 방지하기 위해 좌표 계산 시 박스의 반경(radius)이나 높이/너비를 엄격히 고려합니다.
- **여백(Margin) 활용**: 박스와 화살표 시작점 사이에 최소 2~5px의 여백을 두어 시각적으로 분리된 느낌을 줍니다.

### 4. 애니메이션 및 시각적 밀도
- **일관된 흐름**: 시스템 전체의 데이터 흐름에는 동일한 애니메이션 패턴(예: Glowing Packet)을 적용하여 시각적 통일성을 유지합니다.
- **그래프 밀도**: 'Knowledge Graph'나 'Network'와 같은 노드 시각화 시, 단순히 3~4개가 아닌 다수의 노드(7~10개 이상)와 복합적인 연결선을 사용하여 전문적인 데이터 밀도를 표현합니다.
- **헤더 스타일링**: 박스 상단 헤더에 사선 패턴(`pattern`)이나 대비되는 배경색을 적용하여 영역을 명확히 구분합니다.

## 사용 예시
